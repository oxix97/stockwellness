# 트랙: 포트폴리오 AI 리밸런싱 어드바이저

## 개요
이 트랙은 사용자의 현재 보유 종목을 분석하여 주기적인 리밸런싱 권고안을 제공하는 AI 기반 포트폴리오 어드바이저 기능을 도입합니다. 어드바이저는 시장 데이터 및 기술적 지표를 함께 분석하여 위험, 수익률, 다각화를 최적화할 수 있는 조정 방안을 제안합니다.

## 기능 요구사항
- **데이터 통합**:
    - 현재 포트폴리오 보유 현황 조회 (종목 티커, 수량, 평가 금액).
    - 시장 벤치마크 데이터 접근 (KOSPI, S&P 500 상대 수익률 등).
    - 포트폴리오 내 각 종목의 과거 기술적 지표(RSI, 이동평균선, MACD 등) 조회.
- **AI 분석 엔진**:
    - Spring AI/OpenAI 기반 AI 에이전트 구현.
    - 설정된 목표 비중(Target Weights) 또는 최적 섹터 비중과 현재 비중 비교 분석.
    - 위험 지표(MDD, 변동성 등) 분석을 통한 최적화 기회 식별.
    - 기술적 지표 추세를 활용한 조정 타이밍 제안 (예: "과매수 상태이므로 일부 매도", "과매도 상태이므로 추가 매수").
- **보고서 생성**:
    - AI의 리밸런싱 논리를 설명하는 상세 텍스트 보고서 생성.
    - 각 제안 작업(매수/매도/보유)에 대한 구체적인 근거 포함.
- **스케줄링**:
    - 주간 또는 월간 등 주기적으로 어드바이저 보고서를 자동 생성하는 트리거 구현.

## 비기능 요구사항
- **성능**: AI 보고서 생성은 30초 이내에 완료되어야 함.
- **정확성**: 배치 시스템에 의해 처리된 가장 최근의 EOD(장마감) 데이터를 기반으로 분석해야 함.
- **확장성**: 스케줄링된 시간대에 다수 사용자의 AI 요청을 동시에 처리할 수 있어야 함.

## 수락 기준
- [ ] AI가 보유 데이터 및 시장 데이터를 성공적으로 조회함.
- [ ] 리밸런싱 전략을 요약한 텍스트 보고서가 생성됨.
- [ ] 보고서에는 포트폴리오 내 최소 3개 종목(또는 전체 종목)에 대한 구체적인 분석 근거가 포함됨.
- [ ] 스케줄링된 트리거가 분석을 정상적으로 시작하고 보고서를 저장함.
- [ ] DB -> AI -> 보고서로 이어지는 전체 데이터 흐름을 검증하는 통합 테스트 완료.

## 제외 범위
- 실시간 "매수/매도" 주문 실행 (조언 제공에 한함).
- 외부 증권사 계좌 연동을 통한 자동 매매.
- 텍스트 보고서 내 시각적 차트 (텍스트 기반 통찰로 제한).
