# Track Specification: 핵심 시세 및 차트 데이터 엔진 구축

## 1. 개요 (Overview)
Stockwellness 서비스의 근간이 되는 **핵심 시세 데이터 조회 및 수익률 계산 엔진**을 구축합니다. 야간 배치를 통해 전처리된 수정 주가 데이터를 기반으로, Redis 캐싱과 Java 21의 메모리 내 집계(In-memory Aggregation)를 활용하여 고성능·저부하 차트 API를 제공하는 것을 목표로 합니다.

## 2. 기능적 요구사항 (Functional Requirements)
- **최신 종가 및 기본 정보 조회 (13번):** 특정 종목의 가장 최근 영업일 기준 종가, 등락률, 거래량 등을 조회합니다.
- **수정 주가 기반 차트 데이터 제공 (14, 25번):** 
    - 일간(Daily) 데이터를 기반으로 1주, 1개월, 3개월, 1년, 3년, 5년, 전체 기간의 OHLCV 데이터를 반환합니다.
    - 배치를 통해 미리 계산된 `adjClosePrice`를 사용하여 액면분할 등이 반영된 정확한 차트를 제공합니다.
- **기간별 수익률 계산 (15번):** 설정된 기간(예: 1년, YTD) 동안의 보유 수익률을 계산하여 제공합니다.
- **메모리 내 주/월 집계 (23번):** 일간 데이터를 기반으로 Java 21 Stream API를 사용하여 주봉(Weekly), 월봉(Monthly) 데이터를 실시간 생성합니다.
- **지수 대비 상대 수익률(Benchmark) 비교:** 종목 데이터와 함께 KOSPI/S&P 500 등 벤치마크 지수의 동기화된 수익률 데이터를 함께 제공합니다.

## 3. 기술적 요구사항 (Non-Functional Requirements)
- **배치 전처리 (Pre-calculation):** 수정 주가는 조회 시점이 아닌 배치 시점에 계산되어 DB에 저장되어야 합니다.
- **계층적 캐싱 (Redis):** `price:daily:{ticker}:{year}` 키 구조를 사용하여 연도별 데이터를 캐싱하고, 캐시 미스 시에만 DB에 접근합니다.
- **고성능 집계:** DB의 `GROUP BY` 대신 애플리케이션 레벨의 메모리 연산을 활용하여 DB I/O 부하를 최소화합니다.
- **Hexagonal Architecture:** 시세 조회 도메인 로직을 외부 인프라(DB, Redis)와 분리하여 구현합니다.

## 4. 수락 기준 (Acceptance Criteria)
- [ ] 특정 종목의 1년 치 일봉 데이터를 요청했을 때, 수정 주가가 반영된 데이터가 100ms 이내에 반환되어야 함.
- [ ] 동일한 요청이 반복될 때 Redis 캐시 히트가 정상적으로 발생해야 함.
- [ ] 주봉/월봉 요청 시 일봉 데이터를 기반으로 정확하게 시가/고가/저가/종가가 집계되어야 함.
- [ ] 벤치마크 지수와의 수익률 비교 수치가 정확하게 일치해야 함.

## 5. 범위 제외 (Out of Scope)
- 장중 실시간 시세 처리 (EOD 데이터만 취급)
- 캔들 패턴 탐지 및 가격 갭 분석 (추후 트랙으로 분리)
