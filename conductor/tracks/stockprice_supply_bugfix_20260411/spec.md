# 스펙: StockPriceBatchConfig 투자자 수급 데이터 0 저장 버그 수정

## 개요
현재 주가 수집 배치(`StockPriceBatchConfig`) 실행 시, KIS API(`/uapi/domestic-stock/v1/quotations/inquire-investor`)에서 정상적인 투자자별 수급 데이터가 반환됨에도 불구하고, 데이터베이스(`InvestorSupplyDemand`)에는 대부분의 값이 `0`으로 잘못 저장되는 문제가 발생하고 있습니다. 이 버그의 근본 원인을 분석하고 수정하여 정확한 수급 데이터가 저장되도록 하는 것이 목표입니다.

## 기능 요구사항 (버그 수정 내용)
- `StockPriceBatchConfig` (또는 관련 Reader/Processor/Writer) 내에서 KIS API의 응답 데이터를 파싱하고 엔티티로 매핑하는 로직을 점검하고 수정합니다.
- 배치 실행 시, API에서 제공하는 투자자별(개인, 외국인, 기관 등) 매수/매도/순매수 데이터가 누락되거나 0으로 초기화되지 않고 정확하게 데이터베이스에 반영되어야 합니다.

## 비기능 요구사항
- 배치 프로세스의 성능(실행 시간, 메모리 사용량 등)이 이전과 비교하여 눈에 띄게 저하되지 않아야 합니다.

## 인수 조건 (Acceptance Criteria)
- **자동화 테스트:** KIS API의 응답을 모의(Mocking)하여, 응답 데이터가 엔티티로 올바르게 변환되고 저장되는지 검증하는 단위/통합 테스트가 작성되어 통과해야 합니다.
- **수동 DB 확인:** 로컬 환경에서 실제 배치를 실행한 후, 데이터베이스의 `InvestorSupplyDemand` 관련 컬럼 값들이 정상적으로 적재되었는지 수동으로 확인해야 합니다.
- **API와 비교 검증:** 배치 실행 결과 적재된 데이터가 KIS API 응답(예: Postman 또는 curl 결과)과 일치하는지 비교하여 검증합니다.
- 관련 프로세스 중 에러 로그 발생 여부를 확인하여 조용한 실패(Silent failure) 가능성을 제거합니다.

## 범위 외 (Out of Scope)
- 주가 수집 외의 다른 배치 프로세스 기능 수정
- 수급 데이터를 활용하는 프론트엔드 또는 API 엔드포인트 수정 (데이터 적재까지만 책임짐)
