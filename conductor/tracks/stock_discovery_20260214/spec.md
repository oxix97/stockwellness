# Specification: Stock Discovery & Basic Inquiry (Track A)

## 1. Overview
사용자가 티커 또는 한국어 종목명을 통해 원하는 주식을 빠르고 정확하게 찾을 수 있도록 통합 검색 기능을 제공합니다. 검색 성능 최적화를 위해 Redis와 인덱싱 전략을 활용하며, 검색 편의를 위한 최근 검색어 및 24시간 인기 검색어 기능을 구현합니다.

## 2. Functional Requirements
- **통합 검색 (A-1)**
  - 검색 대상: 티커(Ticker) 및 한국어 종목명.
  - 정렬 기준: 검색어와의 정확도(Accuracy)가 높은 순서대로 상단 노출.
  - 기술 스택: QueryDSL을 사용하여 동적 검색 쿼리 구현.
- **최근 검색어 (A-3)**
  - 저장소: Redis를 활용하여 사용자별 최근 검색 기록 관리 (빠른 읽기/쓰기 및 자동 만료 지원).
  - 기능: 최근 검색 기록 조회 및 개별 항목 삭제.
- **인기 검색 순위 (A-4)**
  - 집계 기준: 최근 24시간(일간) 동안의 검색 로그를 기반으로 상위 10개 종목 산출.
- **검색 결과 편의 (A-5, A-12)**
  - 페이징 처리: 대량 검색 결과 반환 시 성능을 위해 페이징(Pagination) 적용.
  - 종목 상세 메타데이터: 시장(KOSPI/KOSDAQ 등) 및 섹터 정보 포함.
  - 신규 상장 종목: 최근 등록된 신규 종목 리스트 조회 지원.

## 3. Non-Functional Requirements
- **Performance**: 검색 속도 극대화를 위해 검색 대상 필드에 대한 DB 인덱스(커버링 인덱스 등) 전략 수립.
- **Scalability**: Redis를 사용하여 최근/인기 검색어 처리 부하를 메인 DB에서 분리.
- **Reliability**: Spring Batch 또는 스케줄러를 활용하여 24시간 주기 인기 검색어 데이터 갱신.

## 4. Acceptance Criteria
- [ ] 티커 또는 한국어 종목명 입력 시 정확도순으로 검색 결과가 반환되는가?
- [ ] 검색 결과가 설정된 페이징 크기에 맞춰 정상적으로 반환되는가?
- [ ] 최근 검색어가 Redis에 사용자별로 저장되고, 조회 및 삭제가 가능한가?
- [ ] 24시간 집계 주기에 따라 인기 검색어 Top 10이 정확하게 갱신되는가?
- [ ] 검색 대상 필드에 적절한 인덱스가 적용되어 쿼리 성능이 확보되었는가?

## 5. Out of Scope
- 초성 검색 (A-2) 및 마켓/상태별 복합 필터링 (A-6, A-7, A-11)
- 섹터별 전체 종목 리스트 (A-9) 및 프리미엄 트래킹 (A-10)
- 상장 폐지 이력 상세 조회 (A-8)
