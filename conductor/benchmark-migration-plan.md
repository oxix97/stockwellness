# Plan: BenchmarkPrice Schema Migration

## Objective
`BenchmarkPrice` 도메인 엔티티를 영속화하기 위한 PostgreSQL 테이블 스키마를 생성합니다.

## Key Files & Context
- **Entity**: `stockwellness-core/src/main/java/org/stockwellness/domain/stock/price/BenchmarkPrice.java`
- **Migration Path**: `stockwellness-core/src/main/resources/db/migration/`

## Implementation Steps

### 1. Create Migration SQL
`V3__create_benchmark_price.sql` 파일을 생성하여 다음 스키마를 정의합니다.
- `id`: BIGINT GENERATED BY DEFAULT AS IDENTITY (PK)
- `created_at`, `updated_at`: 타임스탬프 (AbstractEntity 대응)
- `ticker`: VARCHAR(20) (NOT NULL)
- `base_date`: DATE (NOT NULL)
- `close_price`: NUMERIC(19, 4) (NOT NULL)
- 기타 가격 필드: NUMERIC(19, 4)
- `volume`: BIGINT
- **Indices**:
    - UNIQUE INDEX: `idx_benchmark_ticker_date (ticker, base_date)`
    - INDEX: `idx_benchmark_date (base_date)`

### 2. Entity Mapping Verification
- 엔티티의 `@Table` 및 `@Index` 선언이 생성된 SQL과 일치하는지 최종 확인합니다.

## Verification & Testing
- `./gradlew :stockwellness-core:test`를 실행하여 Flyway 마이그레이션이 정상적으로 수행되는지 확인합니다.
- H2 및 PostgreSQL(Docker) 환경에서 테이블 생성 여부를 검증합니다.
